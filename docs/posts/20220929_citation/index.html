3 initial setup citation

Remark: In the following setup the bibliography is maintained in a references.bib file and there is no need to add bibliography: references.bib to the YAML medadata field of the document.

  1. add bibliography: references.bib``to_quarto.yml`
  2. add to root directory of the repository a references.bib file with content
---
title: References
---

::: {#refs}
:::
  1. add to root directory of the repository a references.bib file and add (or any
@article{knuth84,
  author = {Knuth, Donald E.},
  title = {Literate Programming},
  year = {1984},
  journal = {Comput. J.},
  pages = {97–111}
}
  1. test by simply put @knuth84 in a blog document and call quarto preview for example from a conda prompt.

remark: Here knuth84 is the unique identifier in the citation database references.bib

4 workflow

4.1 Bibliography Generation

Supose you want to add a new reference to your citation database references.bib, e.g. (Trotta et al. 2022). I usually do the following:

  1. open google scholar
  2. search for your reference, e.g. “trotta routing on a ring”
  3. use the nice link “citation” below the hit of google scholar and choose BibTex next
  4. simply copy and paste the content to your references.bib file. The copied content will look like:
@article{trotta2022pickup,
  title={Pickup and delivery problems with autonomous vehicles on rings},
  author={Trotta, Manuel and Archetti, Claudia and Feillet, Dominique and Quilliot, Alain},
  journal={European Journal of Operational Research},
  volume={300},
  number={1},
  pages={221--236},
  year={2022},
  publisher={Elsevier}
}

4.2 Add a citation in a blog document

  • Markdown format for citation is [@citation]
  • Given example: [@trotta2022pickup] renders to (Trotta et al. 2022)
  • There are other markdown format for citation, e.g. @trotta2022pickup renders to Trotta et al. (2022). For more information just have a look at the sources below.
